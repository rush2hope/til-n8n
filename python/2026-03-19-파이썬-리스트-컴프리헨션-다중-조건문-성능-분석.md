### Context
파이썬에서 대량의 데이터를 필터링할 때 리스트 컴프리헨션(List Comprehension) 내에 여러 개의 조건문(`if`)을 중첩하여 사용하는 방식과 `filter()` 함수를 사용하는 방식 사이에서 가독성과 성능의 트레이드오프가 발생합니다. 특히 `[x for x in data if cond1 if cond2]`와 같은 형태는 조건이 늘어날수록 코드 파악이 어려워지며, 이를 해결하기 위한 기술적 선택 기준을 정리합니다.

### Core
리스트 컴프리헨션의 다중 조건문은 논리적으로 `and` 연산과 동일하게 작동하며, `filter` 함수와 람다(lambda)를 조합한 방식보다 실행 속도가 빠릅니다.

* **리스트 컴프리헨션 방식 (권장 성능)**
```python
# 다중 if 조건문 사용
result = [x for x in range(1000000) if x % 2 == 0 if x % 5 == 0]
```

* **filter() 함수 방식 (함수형 스타일)**
```python
# filter와 lambda 조합
result = list(filter(lambda x: x % 2 == 0 and x % 5 == 0, range(1000000)))
```

### Insight
파이썬의 리스트 컴프리헨션이 `filter()`와 람다 조합보다 빠른 결정적인 이유는 함수 호출 오버헤드(Function Call Overhead)에 있습니다. 리스트 컴프리헨션은 파이썬 인터프리터 수준에서 최적화된 C 루프로 실행되지만, `filter(lambda ...)`는 반복마다 람다 함수를 호출해야 하므로 스택 프레임 생성 및 제거 비용이 추가됩니다.

* **기술적 검증**
* 리스트 컴프리헨션은 내부적으로 `LIST_APPEND` 바이트코드를 사용하여 리스트를 구축하며, 조건문은 인라인으로 처리됩니다.
* `filter`는 이터레이터(Iterator)를 반환하므로 메모리 효율성 면에서는 유리할 수 있으나, 이를 다시 `list()`로 변환하는 과정에서 컴프리헨션보다 느려지는 경향이 있습니다.
* 가독성 측면에서 조건이 3개 이상 늘어날 경우, 컴프리헨션을 여러 줄로 나누어 작성하거나 별도의 필터링 함수를 정의하여 가독성을 확보하는 것이 좋습니다.
* 비전공자나 협업이 중요한 프로젝트에서는 명확한 `for` 루프와 `if` 문을 사용하는 것이 유지보수에 유리하며, 성능이 극도로 중요한 대량 데이터 처리 시에만 컴프리헨션을 최적화 도구로 활용해야 합니다.
