### Context
n8n은 노드 기반의 워크플로우 자동화 도구로, 복잡한 API 연동 및 비즈니스 로직을 시각적으로 설계할 수 있게 지원합니다. 첫 번째 워크플로우를 구축하며 트리거(Trigger)와 액션(Action)의 관계를 이해하고, 데이터가 노드 사이를 어떻게 흐르는지 파악하는 것이 기술적 성장의 핵심입니다.

### Core
n8n 워크플로우는 데이터 아이템(Items)의 배열을 처리하는 방식으로 작동합니다. 각 노드는 입력 데이터를 받아 특정 작업을 수행한 후 다음 노드로 결과 값을 JSON 형태로 전달합니다.

*   **트리거 노드(Trigger Node)**: 워크플로우의 실행 시점을 결정하는 시작점입니다. 일정 예약(Schedule), 외부 시스템의 호출(Webhook), 특정 앱의 이벤트 감지 등이 해당됩니다.
*   **액션 노드(Action Node)**: 데이터를 변환하거나 외부 API로 전송하는 실행 단위입니다. (예: HTTP 요청, 데이터베이스 삽입, 메시지 전송 등)
*   **표현식(Expressions)**: `{{ $json.field_name }}`와 같은 문법을 사용하여 이전 노드에서 발생한 데이터를 동적으로 참조하여 변수화합니다.

```json
{
  "nodes": [
    {
      "parameters": {
        "rule": {
          "interval": [{ "field": "minutes", "minutesInterval": 60 }]
        }
      },
      "name": "Schedule Trigger",
      "type": "n8n-nodes-base.scheduleTrigger",
      "position": [100, 200]
    },
    {
      "parameters": {
        "method": "POST",
        "url": "https://api.example.com/v1/notify",
        "bodyParameters": {
          "parameters": [
            { "name": "message", "value": "워크플로우 실행 성공" }
          ]
        }
      },
      "name": "HTTP Request",
      "type": "n8n-nodes-base.httpRequest",
      "position": [320, 200]
    }
  ],
  "connections": {
    "Schedule Trigger": {
      "main": [[{ "node": "HTTP Request", "type": "main", "index": 0 }]]
    }
  }
}
```

### Insight
n8n의 아키텍처는 모든 데이터를 JSON 객체의 배열로 관리하므로 복잡한 데이터 구조도 유연하게 처리할 수 있습니다. 특히 실행 데이터(Execution Data) 확인 기능을 통해 각 노드 단계별로 입력값과 출력값을 즉시 비교할 수 있어 디버깅 효율성이 매우 높습니다. 초기 단계에서는 간단한 트리거와 알림 노드를 연결하는 것부터 시작하여, 점차 조건문(If)이나 루프(Split In Batches) 노드를 추가하며 로직의 복잡도를 높여가는 접근 방식이 효과적입니다.
